

Область действия и время жизни имен.
Имя объекта программы имеет область 
действия и существует в определенное время.

Область действия - фрагмент кода, в котором объкт известен, или действует описание.
Время жизни - время выполнения программы, в течение которого объект фактически занимает место в памяти.
Объекты - именованные костанты, переменные, функции.

Классификация объектов (статических) по 
области действия:
1. Локальные - объявлены в блоке 
{
   Тип имя;
}
   Область действия - код блока.
   Время жизни - время выполнения блока 
(функции).
2. Глобальные - объявлены вне тела всех функций на уровне медуля.

int x;

void main(void)
{

}  
   Область действия - файл от момента описания до конка файла.
   Время жизни - время выполнения модуля, в котором он описан.
 
Внимание!
ДЛя параметров функций.
1. Локальные данные - параматры по значению.
   Порождены функцией, живут только в теле функции.
2. Глобальные данные - параметры по ссылке.
   Порождены вне функции, область определена вне функции, как и время жизни.


Принцип локализации имен.
Вопрос?
Коллизия имен - имена локального и глобального объект одинаковы.
Зназение локального данного на время выполнения блокатела функции перекроет значение глобального данного, оно перестанет быть известным функции.
По завершении работы функции глобальное данное получит прежнее значение.


Классы памяти.
Любой объект приписан к определенному классу, по умолчанию - auto.

auto - автоматически определены механизмы локализации, как 

static - средство глобализации объекта из тела функции.

register- данное в регистры процессора, является рекомендательным.

Синтакстс:

static Тип Имя;

Как глобальное имя имеет мало смысла.

В теле функции
  Объект получает память один раз при старте программы.
  Обнуляется.
  При повторных входах в функцию значение сохраняется. 
  
  Внешние данные.
Межмодульное взаимодействие.
Данные модуля локальны.Для глобализации данного extern.
